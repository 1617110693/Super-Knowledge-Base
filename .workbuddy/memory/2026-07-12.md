# 2026-07-12 工作日志

## 侧边栏折叠模式滚动条修复 (Sidebar.vue)

**问题**: 应用侧边栏收起后(52px宽)，上下分栏(KB列表/最近对话)的滚动条不工作；"概览"和"新对话"按钮未固定，会随内容滚动/被压缩。

**根因**:
1. `.sc-middle` / `.sc-top-half` / `.sc-bottom-half` 有冗余 `height: 0`，与 `flex: 1 1 0` 的 `flex-basis: 0` 叠加，可能干扰 flex 高度计算
2. 概览按钮(`.sb-nav-btn`)和新对话按钮(`.sb-nav-btn-chat-icon`)缺少 `flex-shrink: 0`，内容多时被压缩变形
3. `.sc-top-half` / `.sc-bottom-half` 缺少 `overflow: hidden`，溢出内容可能泄漏
4. 全局滚动条 6px 在 52px 侧边栏中偏宽

**修复**:
- 移除三处 `height: 0`（`flex-basis: 0` 已足够）
- 给 `.sc-top-half` / `.sc-bottom-half` 加 `overflow: hidden`
- 新增规则 `.sc-top-half > .sb-nav-btn, .sc-bottom-half > .sb-nav-btn-chat-icon { flex-shrink: 0; }` 固定按钮
- 为 `.sc-kb-list` / `.sc-recent-chats` 添加 3px 窄滚动条样式 + `scrollbar-width: thin`

**关键布局结构**（折叠模式）:
```
.sc-middle (flex:1 1 0, overflow:hidden)
├── .sc-top-half (flex:1 1 0, overflow:hidden)
│   ├── button.sb-nav-btn [概览, flex-shrink:0 固定]
│   └── .sc-kb-list (flex:1 1 0, overflow-y:auto) ← 滚动
└── .sc-bottom-half (flex:1 1 0, overflow:hidden)
    ├── button.sb-nav-btn-chat-icon [新对话, flex-shrink:0 固定]
    └── .sc-recent-chats (flex:1 1 0, overflow-y:auto) ← 滚动
```

**文件**: `apps/desktop/src/components/layout/Sidebar.vue`

## 侧边栏折叠模式滚动条深入排查（第二轮）

**用户反馈**: 修改后滚动条仍未出现。

**浏览器实测（Playwright + Chromium 149）**:
- 注入 20 个测试按钮后，`.sc-kb-list` scrollHeight=758 > clientHeight=285 → **canScroll: true**，滚动功能正常
- `.sc-recent-chats` scrollHeight=619 > clientHeight=283 → **canScroll: true**
- 概览/新对话按钮 `flex: 0 0 auto`（flex-shrink:0 生效），固定不滚动 ✓
- CSS 布局修复正确，滚动逻辑无误

**发现的额外问题**:
1. **`recentConversations.slice(0, 8)` 限制** — 对话列表最多只显示 8 个，而容器能容纳约 9 个（283px / 31px ≈ 9.1），对话列表几乎永远不会溢出 → **已移除 slice 限制**
2. **`::-webkit-scrollbar` 在 Chromium 149 不生效** — 即使 `!important` 全局注入，滚动条宽度仍为 15px（系统默认），无法通过 CSS 自定义宽度。但 `scrollbar-color` 标准属性生效
3. `scrollbar-width: thin` 在 Chromium 149 中测试也未改变宽度（返回 10px，和 auto 一样）

**第二轮修复**:
- 移除模板中的 `.slice(0, 8)` 限制
- `scrollbar-width` 从 `auto` 改为 `thin`
- 合并重复的 `.sc-kb-list` / `.sc-recent-chats` 规则
- 保留 `::-webkit-scrollbar` 样式（Tauri WebView2 可能使用不同版本，或许生效）

**关键结论**: CSS 滚动布局是正确的，滚动功能正常工作。滚动条"看不见"的可能原因：
1. 用户数据量不足（KB < 8 个或对话 < 9 个）— 移除 slice 后对话列表能溢出了
2. Windows WebView2 可能使用 overlay scrollbar（仅滚动时显示）
3. 用户需重新构建 Tauri 应用（`npm run tauri dev`）才能看到修改生效

## 侧边栏折叠模式滚动条根因修复（第三轮 — 真正的根因）

**用户反馈**: "条目越多越密集越小，而不是变成滚动模式" — 明确指出按钮被压缩而非触发滚动。

**根因**: `.sc-kb-list` 和 `.sc-recent-chats` 是 `display: flex; flex-direction: column;` 容器，其中按钮的默认 `flex-shrink: 1` 导致内容溢出时**按钮被压缩**（从 36px 压到 22.5px），而不是保持固定大小触发 `overflow-y: auto` 滚动。

**之前遗漏**: 只给概览/新对话按钮加了 `flex-shrink: 0`，但**列表内的按钮没有加**。

**修复**: 新增 CSS 规则：
```css
.sc-kb-list > button,
.sc-recent-chats > button {
  flex-shrink: 0;
}
```

**Playwright 验证结果**（注入 15 个按钮，data-v 属性模拟 Vue scoped CSS）:
| 列表 | scrollHeight | clientHeight | canScroll | 按钮高度 | flex-shrink |
|---|---|---|---|---|---|
| KB 列表 | 568px | 285px | true ✓ | 36px（保持不变）✓ | 0 ✓ |
| 对话列表 | 464px | 283px | true ✓ | 30px（保持不变）✓ | 0 ✓ |

**额外发现**:
- Vite HMR 有时不会更新 scoped CSS，需要硬刷新页面
- `::-webkit-scrollbar` 在 Chromium 149 中不控制滚动条宽度，但 `scrollbar-color` 标准属性生效
- Vue scoped CSS 中 `> button` 选择器会编译为 `> button[data-v-xxxx]`，对 Vue 渲染的按钮生效

## 侧边栏滚动条 hover 显示/隐藏（第六轮）

**需求**: 滚动条只在鼠标悬停在侧边栏时显示，否则隐藏。折叠和展开模式统一。

**方案**: CSS 变量 + `:hover` 伪类
- 各分区容器独立控制：`.sc-top-half`、`.sc-bottom-half`、`.sb-section` 各自设 `--sb-thumb` 变量
- 默认 `transparent`，`:hover` 时变为 `rgba(144,147,153,0.25)`
- 滚动容器（`.sc-kb-list`、`.sc-recent-chats`、`.sb-section-list`）使用 `var(--sb-thumb)` 继承父分区变量
- 效果：鼠标在上半边只显示KB列表滚动条，在下半边只显示对话列表滚动条，展开模式每个section独立

**额外**: 给展开模式的 `.sb-section-list` 也添加了 4px 细滚动条样式（之前用全局 6px），并添加 `overflow-x: hidden`

**演进**: 最初用 `.sidebar:hover` 全局控制（鼠标进侧边栏所有滚动条都显示），用户反馈不够精细，改为各分区独立 `:hover`

## 标签页与对话滚动四项改进（第七轮）

### 1. 标签页滚动条可拖动（TitleBar.vue）
- `onDrag` 中检测鼠标是否在 `.tab-list` 水平滚动条区域（`clientY > rect.bottom - scrollbarHeight`），是则不触发窗口拖拽
- 保留标签页空白区域的窗口拖拽功能

### 2. 标签页拖拽交换位置（TabBar.vue）
- `.tab-item` 添加 `draggable="true"` + `dragstart/dragover/drop/dragend` 事件
- `dragOverPos` 根据鼠标在标签页左半/右半侧决定插入位置（左边缘/右边缘指示线）
- 调用 `tabStore.reorderTabs(tabs)` 更新顺序
- CSS: `.dragging` 半透明，`.drag-over-left/right` 显示蓝色指示线

### 3. 标签页智能压缩（TabBar.vue CSS）
- `.tab-item` 从 `flex-shrink: 0; min-width: 0` 改为 `flex-shrink: 1; min-width: 44px`
- 空间不足时先压缩 `.tab-title`（flex:1, min-width:0），标题被截断
- 压缩到极限（只显示图标）后才触发 `overflow-x: auto` 显示滚动条

### 4. ChatPage 自动滚动智能化（ChatPage.vue）
- 新增 `lastScrollTop` 跟踪滚动方向
- `onScroll` 中用 `scrollDelta = scrollTop - lastScrollTop` 判断方向
- 只有用户**主动向上滚动**（`scrollDelta < -5 && dist > 30`）才取消 autoScroll
- 程序滚动（scrollToBottom）和新内容增长不会误触取消
- `dist < 4`（接近底部）时重新开启 autoScroll
